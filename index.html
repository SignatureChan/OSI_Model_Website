Layer 5: Session Layer

The Session Layer is responsible for establishing, managing, and terminating communication sessions between applications on different devices. It works together with Layer 4 of the OSI model by ensuring a persistent connection between the applications so that data can be properly transported.

Main Functions

Session Setup

+

Establishes, or sets up, a communication session between two devices to coordinate the initial connection and prepare for data transfer.

Session Management

+

Manages the session through dialogue control, which coordinates data exchange to ensure data does not collide and remains orderly during transmission.

Session Synchronization

+

Creates checkpoints in data streams that serve as recovery points. If the session is interrupted, data transfer can resume from the last checkpoint instead of starting over, which prevents data loss.

Session Termination

+

Once all data has been exchanged, the Session Layer gracefully closes the session, ending the conversation and releasing resources like memory and connection states.

Examples of Protocols

SIP

Session Initiation Protocol

Initiates, manages, and terminates sessions over the internet for voice and video calls.

H.245

Multimedia Protocol

Performs the same functions but specifically for multimedia communication sessions.

ASP

AppleTalk Session Protocol

Performs the same functions but specifically for Apple Devices.

More Information

How it Coordinates Ongoing Data Exchanges

+

It coordinates ongoing data exchanges using these functions to ensure that established conversations are structured, synchronized, and recoverable to prevent data loss and communicate reliably.

Relationship to Other Layers

+

Layer 4 (Transport): Layer 5 builds on the Transport Layer by using it as a foundation to hold the session. Layer 4 provides the services and resources, and Layer 5 uses these services to establish and manage the session.

Layer 6 (Presentation): Layer 6 ensures that data passed through the session is in an appropriate format to be either passed up to Layer 7 to be presented or down to Layer 4 to be transported.

Layer 4: Transport Layer

The Transport Layer ensures reliable end-to-end data communication by managing data segmentation, reassembly, error detection, and flow control. It essentially serves as a bridge between the upper layers (Layers 5-7) and the lower layers (Layers 1-3), connecting applications with the underlying network infrastructure.

Main Functions

Segmentation

+

Breaks data into smaller chunks called segments for transmission and adds headers. The headers can contain source/destination port numbers, sequence numbers, checksums, window size, and acknowledgement numbers to ensure proper delivery in the right order.

Reassembly

+

Reassembles segments into a complete data message at the destination using sequence numbers from the header, which ensures that data arrives in the correct order.

Error Checking

+

Uses checksums in headers that are calculated by the sender and recalculated by the receiver. If the checksums don't match, the data is considered corrupt.

Flow Control

+

Prevents a fast sender from overwhelming a slow receiver by adjusting data transmission speed. The receiver tells the sender how much data it can handle, and the sender adjusts dynamically. However, this is only implemented in TCP, not UDP.

Examples of Protocols

TCP

Transmission Control Protocol

A protocol that guarantees reliable, ordered delivery by establishing a connection a three-way handshake (SYN → SYN-ACK → ACK). TCP supports flow control and ensures all data arrives correctly.

UDP

User Datagram Protocol

A simple and fast protocol that sends data without establishing a connection first. It does not guarantee the delivery of all packets in order, and cannot perform flow control. UDP prioritizes speed over reliability.

SCTP

Stream Control Transmission Protocol

A hybrid protocol that combines features of TCP and UDP. It is less commonly used due to its lack of implementation and support in systems.

More Information

Real-World Examples

+

Sending an email uses TCP because it is important that the data is completely intact and arrives at the correct destination in order. However, online gaming uses UDP because speed is more prioritized than accuracy.

Relationship to Other Layers

+

Layer 5 (Session): Layer 4 provides the transport service (TCP or UDP) that Layer 5 relies on to maintain consistent connections between applications. Layer 5 then establishes, manages, synchronizes, and terminates the connection defined by Layer 4.

Layer 3 (Network): Layer 4 receives routed packets from Layer 3 and ensures they're delivered to the right application by segmenting or reassembling them using port numbers.
