Layer 5: Session Layer

The Session Layer is responsible for establishing, managing, and terminating communication sessions between applications on different devices. It works together with Layer 4 of the OSI model by ensuring a secure and persistent connection between the applications so that data can be properly transported.

üîß Main Functions

Session Setup

+

The Session Layer establishes a communication session between two devices, coordinating the initial connection and preparing for data exchange.

Session Maintenance

+

Manages the session through dialogue control, coordinating data exchange to ensure data does not collide and remains orderly during transmission.

Synchronization

+

Creates checkpoints in data streams that serve as recovery points. If the session is interrupted, data transfer can resume from the last checkpoint instead of starting over, preventing data loss.

Session Termination

+

Once all data has been exchanged, the Session Layer gracefully closes the session, ending the conversation and releasing resources like memory and connection states.

Real-World Examples

+

Video calls and live conference streaming rely on the Session Layer to keep connections stable even with temporary interruptions. The checkpoint system ensures that if your internet briefly disconnects during a video call, the session can resume without having to reconnect from scratch.

Layer Relationships

+

With Layer 4 (Transport): Layer 5 builds on the Transport Layer by using Layer 4 as a foundation to hold the session. Layer 4 provides services and resources, and Layer 5 uses these tools to establish and manage the session.

With Layer 6 (Presentation): Layer 5 ensures that data passed through the session is in an appropriate format to be either passed up to Layer 7 to be presented or down to Layer 4 to be transported.

Layer 4: Transport Layer

The Transport Layer ensures reliable end-to-end data communication by managing data segmentation, reassembly, error detection, and flow control. It serves as a bridge between the upper layers (Layers 5-7) and the lower layers (Layers 1-3), connecting applications with the underlying network infrastructure.

üîß Main Functions

Data Segmentation

+

Breaks data into smaller chunks called segments for transmission and adds headers containing source/destination port numbers, sequence numbers, checksums, window size, and acknowledgement numbers to ensure proper delivery and reassembly.

Data Reassembly

+

Reassembles segments into a complete data message at the destination using sequence numbers from the header, ensuring data arrives in the correct order.

Error Checking

+

Uses checksums in headers that are calculated by the sender and recalculated by the receiver. If checksums don't match, the data is considered corrupt and can be retransmitted.

Flow Control

+

Prevents a fast sender from overwhelming a slow receiver by adjusting data transmission speed. The receiver indicates how much data it can handle, and the sender adjusts dynamically. Note: Only implemented in TCP, not UDP.

üì° Examples of Protocols

TCP

Transmission Control Protocol

Guarantees reliable, ordered delivery through connection establishment using a three-way handshake (SYN ‚Üí SYN-ACK ‚Üí ACK). Supports flow control and ensures all data arrives correctly.

UDP

User Datagram Protocol

Fast and simple protocol that sends data without establishing a connection first. Does not guarantee delivery or order, and cannot perform flow control. Prioritizes speed over reliability.

SCTP

Stream Control Transmission Protocol

A hybrid protocol combining features of TCP and UDP. Less commonly used due to limited implementation and system support.

üí° More Information

Real-World Examples

+

Sending an email uses TCP because complete data integrity is critical - every character must arrive correctly and in order. Online gaming uses UDP because speed is more important than perfect accuracy; even if a few packets are lost, the overall experience remains smooth and responsive.

Layer Relationships

+

With Layer 5 (Session): Layer 4 provides the transport service (TCP or UDP) that Layer 5 relies on to maintain consistent connections between applications. Layer 5 then establishes, manages, terminates, and synchronizes the connection defined by Layer 4.

With Layer 3 (Network): Layer 4 receives routed packets from Layer 3 and ensures they're delivered to the correct application by segmenting or reassembling them using port numbers.
